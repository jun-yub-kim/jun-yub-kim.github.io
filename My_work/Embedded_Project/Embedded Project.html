Wireless connection

development process

There is bluetooth, wifi for wireless connection
My client request to use bluetooth at first,


As a result of investigation, almost all bluetooth module for MCU needs pre-setting using 'AT mode'
After completing the master and slave setup, data communication was successful


However, researching the exhibition hall, I expected that there would be noise in Bluetooth due to a large number of visitors.
So the change from Bluetooth to Wi-Fi was inevitable.



First, I opend server / client method for data communication.
But, there was a delay rate.


Optimization was performed according to the esp32 chip used as a module, and a very short delay rate was secured as follows.



Touch sensor


For interaction with MCU chip on LED <-> MCU chip on pebble,
I had to attach a touch sensor, and send the data over Wi-Fi in LED.s


The touch sensor is commonly used for mcu chips.
However, it seemed necessary to have a touch sensor that can be attached to the pebble
according to the purpose of the exhibition.
Micro current flows in human body.
Using electrical resistance and a metal film,
it's possible to make sensor that detects the Micro current flows in human body.

Pinmap was like this.



Since the capacitive part code has more than 400 lines, it has been header-filed so that we could dramatically reduce the readability of the code volume.
